资源定位是feather从fis中继承过来的3种能力之一，在之前的文档中其实已经有使用到，比如url自动添加domain，版本号等，都属于资源定位，资源定位还有一些其他用法，比如：

//配置config，产出的所有的静态资源的baseurl都变为/a/b/c/d
feather.config.set('statics', '/a/b/c/d');	//默认为/static
//   /static/1.js => /a/b/c/d/1.js

<!--编译后变为 /a/b/c/d/1.png-->
<img src="/static/1.png" />
<!--编译后变为 /a/b/c/d/1.js-->
<script src="/static/1.js"></script>

在js中获取一个文件产出后的路径：

var img = new Image;
img.src = __uri('/static/1.png');

产出后的代码变为：

var img = new Image;
img.src = '/a/b/c/d/1.png';

另外，feather对__uri进行了一次扩展，让__uri支持第2个，3个参数，分别为 useHash? useDomain? 默认2个值都为true

var img = new Image;
img.src = __uri('/static/1.png', true, false);	//只加md5值，不加domain值
